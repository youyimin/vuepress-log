# 走近苏宁科技

  • 1914 售后服务管理系统 空调业内第一套
  • 1996 零售开票系统 中国第1张电子增值税票
  • 1999 ERP系统 E连锁，全面使用
  • 2000 ERP管理系统上线 成立信息系统管理中心
  • 2003 全面实施集中式办公自动化（OA）系统
  • 2004 远程多媒体 监控系统上线
  • 2006 SAP/ERP系统上线 世界零售灯塔工程
  • 2008 全面实现SOA的企业架构 构建完成财务共享服务中心
  • 2010 苏宁易购正式上线 徐庄数据中心成立
  • 2013 北京和硅谷研发中心成立，标志着全球研发组织布局初显
  • 2014 零钱宝、大聚惠 闪拍频道等上线
  • 2016 南京自动化仓库上线是 国内乃至国际先进性代表
  • 2018 智慧零售三剑客：小店、零售云、拼购等系统上线
  • 智能化
    • 运营 泰坦，千变，电子发票
    • 供应链 星河，智能选品，智能促销
    • 物流 智能路由时效，智能全程监控，智慧零售仓
    • 支付 一键付，支付决策机器人，智能风控
  • 平台&技术
    • 大数据平台 百川，纵横，天工
    • 云平台 基础云，数据库，中间件
    • 基础设施技术 巡视简图技术，流量分析技术，同城异地多活
    • 智慧零售创新技术 机械臂，移动售货机器人，智能购物车
  • 产品层
    • 采购 鹰眼，智能供应引擎，诸葛大师
    • 销售 千里传音，数据洞察大师，诸葛大师
    • 服务 智能推荐，烽火台，未来GO
    • 物流 天眼，智能物流引擎
    • 业态 门店客流分析，身边的苏宁
  • 苏宁科技集团主要由三大公司和职能支持体系组成

    • L1-L2 需要笔试，另外用户体验、项目管理序列需要参加面试
    • L3-L4 笔试加面试
    • L5及以上 面试
  • PDF申报条件
    • L1-L2 贡献值0
    • L3-L4 贡献值>=10
    • L5-L6 贡献值>=20
    • L7-L10 贡献值>=30
  • 职级和专业能力对应
    • B6 L1（1年）
    • B7 L2 （3年）
    • B8 L3-L4 （L3 5年，L4 6年）
    • B9 L5-L6 （L5 8年，L6 9年）
    • B10 L7-L8 （10年+）
    • B11 L9（10年+）
    • B12 L10（10年+）

# 项目管理规范培训

  • 项目类型分为软件研发类、咨询设计类、基础架构类、管理类项目
  • 立项材料包含项目定义、阶段计划、资源计划、风险应对计划等内容
  • 变更
    1. 变更准备
    2. 变更审核 变更评审委员（CCB组织）
    3. 变更执行
    4. 在提交变更申请之前，项目经理需要组织CCB决策会议，会后需要输出CCB会议纪要。
  • 结项
    • 上线后一个月内发起
  • 项目经理识别风险级别为中、高级别项目风险时，应在当天以邮件形式上报至中心PMO及对应的上级部门主管，并持续跟踪风险。
  • 当原始业务需求的期望完成日期需要变更时，由需求申请人发起
  • 当原始业务需求的计划上线日期需要变更时，由需求产品负责人发起

# ITP

  • 目前部门日志既可以填写工作内容，也可以选择工时。
  • ITP系统中，工作日志类型包含部门日志与项目日志。
  • 项目锁定人员，可以随时提报项目风险。
  • ITP中创建的版本既可以关联单个项目中的需求也可以关联多个项目中的需求。
  • ITP中版本发布只有版本负责人可以发起。
  • 原始业务需求申请
    • 各大产业总部所有人员都可以发起，苏宁易购大区只允许大区正副职发起
  • 自提自解决需求
    • 自提自解决需求为信息自己创建需求，实施完成后，无需到业务方确认
  • 周日报
    • 周/日报提交后，系统会在第二天10点统一定时推送
  • 当信息方完成需求后，发起“需求关闭”流程需要业务方审批
